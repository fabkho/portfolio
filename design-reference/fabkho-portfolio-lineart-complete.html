Technical Memo // 012

Lazy-loading FontAwesome icons in Vue/Nuxt

When architecting a production system like anny.co, bundle size directly impacts operational efficiency. We identified a critical bottleneck during the Nuxt migration: FontAwesome CommonJS icons were inflating the initial payload, causing a 3+ minute build time.

To bypass this, we constructed a mechanism that converts CJS definitions into ESM blocks, allowing the bundler to selectively load only the necessary vectors via the CDN edge.

Execution Code
// fontawesome-esm-converter.ts import { transform } from 'build-tools'; export const lazyLoadIcon = async (iconName) => { const module = await import( /* webpackChunkName: "icon-[request]" */ `@fortawesome/free-solid-svg-icons/${iconName}.js` ); return module.definition; };

Implementing this zero-dependency Vue component slashed our build times radically. The client now requests SVG definitions incrementally as they enter the viewport.
