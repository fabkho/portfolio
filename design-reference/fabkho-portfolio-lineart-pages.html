Lazy-loading FontAwesome icons in Vue/Nuxt

When architecting a production system like anny.co, the bundle size directly impacts operational efficiency. We identified a critical bottleneck during the Nuxt migration: the FontAwesome CommonJS icons were inflating the initial payload, causing a 3+ minute build time and degrading the Time to Interactive (TTI) metrics.

The standard library imports the entire icon definition set into the client bundle. To bypass this, we must construct a mechanism that converts these CJS definitions into ESM blocks, allowing the bundler to selectively load only the necessary vectors via the CDN edge.

Execution Code
// fontawesome-esm-converter.ts import { transform } from 'build-tools'; export const lazyLoadIcon = async (iconName) => { const module = await import( /* webpackChunkName: "icon-[request]" */ `@fortawesome/free-solid-svg-icons/${iconName}.js` ); return module.definition; };

Annotation: By utilizing dynamic imports wrapped in webpack chunk instructions, we force the bundler to emit discrete files.

Implementing this zero-dependency Vue component slashed our build times radically. The client now requests SVG definitions incrementally as they enter the viewport. This is not merely an optimization; it is a fundamental correction of the delivery pipeline.
